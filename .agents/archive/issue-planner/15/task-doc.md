# Task Doc — Phase 1: Foundation — Tailwind 4, Motion, PlayStation design tokens, app shell (#15)

## Prerequisites

- [ ] None — this issue is foundational (blocked by nothing). Work happens entirely in `site/`
      (plus `pnpm-lock.yaml` from new deps).

> The work is small and tightly coupled (deps → config → tokens → shell all in one package),
> so it is **not** phased into separate PRs. The sub-headings below are logical ordering within
> a single change, sequenced so each step compiles on top of the previous.

## Step group 1: Dependencies & build config

- [ ] Add to `dependencies` in `site/package.json`: `tailwindcss`, `@tailwindcss/vite`,
      `motion`, `@fontsource-variable/source-sans-3` (pick versions compatible with Vite 7 /
      React 19; let `pnpm add` resolve latest).
- [ ] Run `pnpm --filter site add tailwindcss @tailwindcss/vite motion @fontsource-variable/source-sans-3`
      (regenerates `pnpm-lock.yaml`).
- [ ] Register the Tailwind plugin in `site/vite.config.ts`: `import tailwindcss from '@tailwindcss/vite'`
      and add `tailwindcss()` to the `plugins` array (keep `react()` and the `test` block intact).

## Step group 2: Design tokens & global styles

- [ ] Create `site/src/styles/theme.css`:
  - [ ] First line `@import "tailwindcss";`
  - [ ] `@theme { … }` block defining every token from the design doc: surface scale
        (`--color-surface-0: #0B0F1A` + elevated glassy layers), hero accent
        (`--color-ps-blue: #0070D1` + hover + glow), shape accents (triangle/circle/cross/square),
        trophy metals (bronze/silver/gold/platinum), a type scale (`--text-*`), radii
        (`--radius-*`), and glow `--shadow-*` tokens.
  - [ ] Global base styles (outside `@theme`, in a `@layer base` or plain rules): dark `body`
        background + text using the surface/foreground tokens, Source Sans 3 Variable font stack,
        PS-blue `:focus-visible` outline/ring, and `::selection` color.

## Step group 3: Wire up entry point & document

- [ ] In `site/src/main.tsx`: import `@fontsource-variable/source-sans-3` and `./styles/theme.css`;
      import `MotionConfig` from `motion/react` and wrap the tree
      (`<MotionConfig reducedMotion="user">` outermost or around `<App/>`).
- [ ] In `site/index.html`: add `<meta name="theme-color" content="#0B0F1A" />` (page surface color) to `<head>`.

## Step group 4: Motion presets

- [ ] Create `site/src/motion/presets.ts` exporting shared variants/transitions:
      `fadeRise` (opacity + y translate), `staggerChildren` (container variant with
      `staggerChildren` transition), and `glowPulse` (repeating box-shadow/opacity pulse),
      using tokenized durations/easings so later work composes these rather than redefining them.

## Step group 5: App shell

- [ ] Create `site/src/components/AppShell.tsx`: a component taking `children`, rendering a
      sticky translucent `<header>` (wordmark text — not a Sony logo — + a `<nav>` of `NavLink`s:
      one per `players` entry via `playerPath`, plus a `Compare` link via `COMPARE_PATH`, each
      with an active-route indicator through `NavLink`'s `isActive`/`aria-current`), followed by
      the page content (`children`). Import `players` from `../config/players` and
      `playerPath`/`COMPARE_PATH` from `../routes` — never hardcode player keys.
- [ ] In `site/src/App.tsx`: wrap the existing `<Routes>…</Routes>` block in `<AppShell>` without
      changing any `<Route>` (route structure must stay identical).
- [ ] Ensure the shell's wordmark does not collide (by accessible name/role) with page `<h1>`s
      that `routes.test.tsx` queries.

## Step group 6: Tests

- [ ] Write `site/src/components/AppShell.test.tsx`: render inside `MemoryRouter`; assert a nav
      link exists for every `players` entry (`getByRole('link', { name: … })` with correct `href`)
      plus a `Compare` link; assert the active link exposes `aria-current="page"` for the current
      `initialEntries` route. Mirror the query style of `site/src/pages/SplashPage.test.tsx`.
- [ ] Confirm `site/src/routes.test.tsx` still passes **unchanged** (do not edit it).

## Pre-Commit Gate

Verification commands come from `AGENTS.md` → `## Commands` → `site/` workspace commands. Run
and confirm all green before committing:

- [ ] `pnpm --filter site lint` ✅
- [ ] `pnpm --filter site typecheck` ✅
- [ ] `pnpm --filter site test` ✅
- [ ] `pnpm --filter site build` ✅
- [ ] (Sanity) `pnpm --filter site dev` and eyeball the shell + dark theme once.
